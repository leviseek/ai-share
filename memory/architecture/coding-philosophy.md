# 编码哲学

## 核心原则

- **最小正确改动**：每次改动只解决当前问题，不做无关重构
- **根因优先**：不表面修补，定位根本原因
- **测试驱动**：先写失败测试，再写最小实现让测试通过
- **渐进增强**：简单方案优先，只在必要时引入复杂度

## 代码质量

- 可读性 > 巧妙性：代码首先是写给人类读的
- 显式优于隐式：不依赖魔法、自动推断和隐式约定
- 防御性编程：处理边界情况和错误路径
- 注释说明意图，不重复描述行为

## 架构原则

- 复用项目已有模式，不另起炉灶
- 分层清晰：orchestration、business logic、IO 分离
- 接口稳定：公共 API 和数据结构的变更需评估影响面
- 依赖克制：不主动引入新依赖；确需时评估必要性、体积、License 和维护状态

## 长期主义

- 配置即代码，可版本控制、可审查
- 知识结构化沉淀，不依赖聊天记录
- 系统可迁移、可复制、可审计
